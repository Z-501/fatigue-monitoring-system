# 测试与验证策略概览

本项目涉及多端协同、算法服务和 Agent 服务，测试策略按链路而不是单一模块组织。

## 1. 后端基础能力验证

- 用户登录、Token 校验与会话续期
- 高频接口限流与异常响应
- Redis/Caffeine 缓存命中、失效与回填
- 分布式锁防击穿流程

## 2. 在线监测链路验证

- Flutter 端设备状态展示与采集状态切换
- 基线校准数据上传与校准结果返回
- EEG 时间窗上传与疲劳预测结果返回
- 疲劳分数、风险等级和趋势状态展示
- 下游 Python 服务异常时的错误封装与前端提示

## 3. 离线实验链路验证

- Web 端创建实验 session
- Flutter App 绑定同一 session
- 双端 ready 与统一倒计时
- 行为事件、trial 结果、主观评分上传
- EEG chunk 分段上传与最终归档

## 4. Agent 链路验证

- 当前疲劳状态、历史趋势、近期干预记录的上下文构造
- RAG/Tool 调用链路
- 状态解释和干预建议生成
- Prompt、上下文、检索结果、模型响应与评估结果记录

## 5. 历史回放链路验证

- 完整回放接口返回 state / history_analytics / context
- 最近窗口接口支持曲线增量刷新
- 风险占比、峰值点、突变点等统计结果可展示
