# 测试与验证策略

本文件展示项目的测试思路，不展开具体测试代码。公开仓库中可放置脱敏后的测试说明、接口场景和演示步骤。

## 测试维度

| 维度 | 说明 |
| --- | --- |
| 后端接口测试 | 验证登录、监测会话、历史查询、Agent 入口等核心接口 |
| 前后端联调测试 | 验证 Flutter / Web 与 Java 后端之间的数据流转 |
| 算法服务联调 | 验证 Java 调用 Python 疲劳识别服务的请求与返回链路 |
| 实时推送测试 | 验证监测状态变化后客户端能否接收推送结果 |
| 实验流程测试 | 验证实验创建、绑定、运行、上传和结束流程是否完整 |
| 演示场景测试 | 使用固定流程验证系统能否稳定完成一次完整演示 |

## 建议展示方式

公开仓库中建议提供：

- 核心流程测试清单。
- 脱敏后的 HTTP 请求示例。
- 演示视频中的操作步骤。
- 关键页面截图和结果截图。

不建议公开真实测试账号、真实实验数据、密钥、模型权重或未脱敏日志。
