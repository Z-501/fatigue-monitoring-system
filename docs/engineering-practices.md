# 工程实践总结

本项目重点展示一个 AI/算法能力如何融入全栈业务系统，而不是单独展示某一个页面或模型脚本。

## 工程实践方向

| 方向 | 实践 |
| --- | --- |
| 模块化 | 将移动端、Web 端、Java 后端、Python 算法服务和 Agent 服务拆分为相对独立模块 |
| 服务编排 | Java 后端统一处理权限、会话、状态、缓存、持久化和外部服务调用 |
| 配置隔离 | 使用环境变量与示例配置区分本地环境、演示环境和敏感配置 |
| 数据沉淀 | 将实时监测结果和实验事件沉淀为历史记录，为回放、报告和 Agent 提供上下文 |
| 实时反馈 | 使用实时推送机制让客户端能够及时感知疲劳状态变化 |
| 文档化 | 通过架构图、流程图、接口概览和演示说明提高项目可读性和展示价值 |

## 展示重点

公开仓库更适合突出以下内容：

1. 系统整体复杂度：多端、多服务、多数据流。
2. 业务闭环完整性：采集、推理、存储、推送、解释。
3. 架构角色清晰：Java 负责业务中枢，Python 负责智能服务。
4. 工程边界明确：展示设计与演示材料，不公开敏感源码和数据。
